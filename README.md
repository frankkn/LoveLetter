# Love Letter 情書網頁版

使用 TypeScript、Vite 與原生 DOM 製作的《情書 (Love Letter)》單頁網頁遊戲。支援手機操作、PWA 安裝、2 至 4 人本機對戰，以及會記憶與推理的 AI 對手。

## 目前功能

- **完整情書牌組規則**
  - 8 種卡牌：衛兵、神父、男爵、侍女、王子、國王、伯爵夫人、公主。
  - 支援抽牌、出牌、棄牌、淘汰、牌堆耗盡比大小、聯賽硬幣計分。
  - 伯爵夫人強制出牌、公主棄牌出局、侍女保護等核心規則皆已實作。

- **2 至 4 人本機對戰**
  - 玩家可選擇對戰電腦數量。
  - 人類玩家出局後，剩餘 AI 會自動繼續對戰直到本局結束。
  - 回合控制器會跳過已出局玩家，避免多人局 softlock。

- **AI 記憶與推理**
  - AI 使用神父、男爵平手、國王交換等資訊後，會記住已知手牌。
  - AI 使用衛兵時會優先鎖定已知目標並精準猜牌。
  - AI 會避免部分自殺式出牌，例如男爵留手衛兵、王子指定自己棄公主等情境。

- **精緻卡牌視覺**
  - 8 種卡牌皆有專屬角色插畫。
  - 卡牌包含名稱、點數、圖片、效果描述與行動提示。
  - 支援出局、保護、勝者、當前回合等視覺狀態。

- **手機友善操作**
  - 手機沒有 hover，因此玩家手牌採用雙階段操作：
    - 第一次點擊：選取卡牌並顯示描述。
    - 第二次點擊同一張卡：確認出牌。
  - 點擊空白區域會取消選取，降低誤觸出牌機率。

- **彈窗與資訊輔助**
  - 目標選擇、衛兵猜牌、神父看牌、男爵對決、王子棄牌、國王交換等互動彈窗會內嵌即時出牌統計。
  - 玩家在做決策時可以同步查看目前已出牌數與剩餘牌況。
  - 神父看牌結果只保留確認按鈕，不提供不合理的取消返回。

- **PWA 支援**
  - 提供 `manifest.json` 與 `sw.js`。
  - 支援 Android Chrome 安裝提示。
  - iOS 加入主畫面後可使用 standalone 顯示模式。
  - 具備基本離線快取。

## 技術棧

- **TypeScript**：遊戲狀態、卡牌效果、AI 邏輯與 UI 流程。
- **Vite**：開發伺服器與前端建置。
- **Vanilla DOM**：不依賴 React/Vue，直接操作 DOM。
- **CSS3**：響應式版面、卡牌視覺、彈窗、手機互動與 PWA 全螢幕體驗。

## 安裝與執行

安裝依賴：

```bash
npm install
```

啟動開發伺服器：

```bash
npm run dev
```

預設網址：

```text
http://localhost:5173
```

建置 production 版本：

```bash
npm run build
```

預覽建置結果：

```bash
npm run preview
```

## PWA 安裝

建議先執行 production build，再使用支援 HTTPS 或 localhost 的環境測試 PWA：

```bash
npm run build
npm run preview
```

Android Chrome 可透過瀏覽器安裝提示加入主畫面。iOS Safari 可透過分享選單的「加入主畫面」安裝。

## 遊戲操作

1. 在主選單選擇本機對戰與電腦人數。
2. 玩家回合先按「抽牌」。
3. 手上有兩張牌時，點擊卡牌查看效果，再次點擊同一張牌出牌。
4. 需要選目標或猜牌時，依彈窗操作。
5. 本局結束後可查看排名，並開始下一局。

## 備註

本專案目前以單機本機對戰為主，AI 與 UI 邏輯都集中在 `src/main.ts`，樣式集中在 `src/style.css`。若後續要擴充線上多人或拆分模組，建議優先將卡牌效果、AI 決策、回合控制器與 DOM 渲染拆成獨立檔案。
